本文件是可直接执行和签字的 UAT 交付物。Codex 只填写可由自动化证明的结果，不代替财务人员判断业务真值或签名。

## 数据规则

1. 只在名称以 `_test` 结尾的 PostgreSQL 数据库执行 UAT。
2. `数据文件/` 只读；不移动、不改名、不覆盖、不删除。
3. 原始文件名、客户名、电话、卡号、地址、金额明细、OCR 原文和完整哈希不得写入本文件、Git、截图或 CI artifact。
4. 样本选择和逐字段真值写入 Git 忽略的 `.realdata-test/uat/`，公开验收单只记录匿名 `sampleId`、结论和问题编号。
5. OCR 结果必须由财务纠错并确认后才能生成经营记录；模型文本不得直接成为财务数字来源。

## UAT 场景

### UAT-01 运输账单 Excel 导入与对账

1. 财务选择一个匿名运输账单样本和目标项目/模板。
2. 检查 Sheet 目录、隐藏状态、表头候选、公式和媒体提示。
3. 明确选择明细 Sheet 与表头范围，逐列确认日期、金额、车牌、司机、起点、终点、票数和吨数。
4. 预览错误、重复、忽略和合法行；确认后核对 `BusinessRecord`、动态字段、来源快照、audit 和 ledger。
5. 将记录总额与财务本地 L3 真值逐分比较。

签字项：入账粒度 `明细 / 汇总 / 其他：____`；字段映射正确 `是 / 否`；金额一致 `是 / 否`；问题编号 `____`。

### UAT-02 考勤/劳务结算入账政策

1. 选择一个匿名考勤或司机结算样本。
2. 财务决定按每日明细、人员月汇总或结算单汇总入账。
3. 验证工时、人员、岗位、单价、金额和期间字段，不把同一份明细与汇总同时记入 actual。
4. 将预算或资金申请标为 `budget`，对账汇总标为 `reconciliation`，实付才进入 `actual`。

签字项：最终粒度 `____`；明细/汇总互斥规则 `____`；负数/冲销方向 `____`。

### UAT-03 报销工作簿与内嵌凭证

1. 选择含报销主表和内嵌凭证的匿名工作簿。
2. 确认表格数据与媒体对象分层处理，图片不变成普通字段。
3. 验证主表记录与原始文件关联，必填凭证缺失时不允许无提示确认。
4. 确认公式不执行，只有财务显式授权时才使用缓存结果。

签字项：主表/凭证归属规则 `____`；凭证缺失处理 `阻断 / 警告 / 其他：____`。

### UAT-04 OCR 纠错与确认

1. 分别选择电子票据、1080x4794 长图和多页凭证匿名样本。
2. 核对页范围、字段证据、页码、置信度和冲突提示。
3. 人工修正日期、金额、主体等字段，检查 before/after、操作者、时间和 audit。
4. 确认前查询经营记录数为 0；确认后只生成一条记录；重复确认不重复入账。
5. 在本地标签文件完成字段真值复核后再计算准确率。

签字项：可接受单据复核时长 `____`；最低准确率 `____`；35 页凭证拆分政策 `____`。

### UAT-05 报表与人工汇总

1. 选择已由财务确认的 L3 记录，按日/月/项目汇总收入、成本和利润。
2. 核对北京时间边界、负数冲销、零金额和空分类。
3. 确认报表只统计 confirmed `actual`，不重复统计 `budget`、`reconciliation` 或汇总源。
4. 金额必须逐分一致；不一致时记录最小复现和责任模块。

签字项：样本记录数 `____`；人工总额/系统总额一致 `是 / 否`；问题编号 `____`。

### UAT-06 老板 AI 标准问题

1. 老板从 72 条标准问题中抽查总额、环比/同比、排行、成本结构、异常、下钻和无数据场景。
2. 每个数字与相同期间的报表工具逐项核对。
3. 确认回答包含期间、口径和来源；无数据不编造；employee/finance/reviewer 无法访问 boss AI。
4. 不关闭 grounding 和受控 renderer。原始 Qwen 输出不直接作为财务结论。

签字项：抽查数量 `____`；数字一致 `是 / 否`；口径可理解 `是 / 否`；问题编号 `____`。

### UAT-07 重复文件与业务重复

当前系统保存 SHA-256、任务幂等键和确认幂等，能阻止同一任务重复确认，但尚未跨 Excel/OCR/手工来源自动判定“同一业务”。按用户决定，本阶段不自动阻断近似重复。

1. 重复上传完全相同文件，财务比较哈希、来源任务和确认预览。
2. 财务决定继续、取消或作废；系统不得自行删除原始证据。
3. 在财务定义业务唯一字段前，人工复核是正式降级方案。

签字项：重复判断字段 `____`；处理方式 `阻断 / 警告 / 允许版本替换`；规则负责人 `____`。
