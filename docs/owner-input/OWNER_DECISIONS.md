## 状态词

- `OWNER_CONFIRMED`：项目负责人已经明确选择，可据此实现产品行为。
- `SAFE_DEFAULT_ACTIVE`：细节尚未确认，系统采用失败关闭的安全默认。
- `ENGINEERING_VERIFIED`：代码和自动化证据已通过，不等于真实业务验收。
- `REAL_SAMPLE_NEEDED`：必须用真实或脱敏真值验证。
- `EXTERNAL_RESOURCE_NEEDED`：必须由项目负责人提供外部环境、账号或资源。
- `OWNER_UAT_VERIFIED`：项目负责人已亲自完成 UAT；当前尚未达到。

## 当前安全默认

| 范围 | 状态 | 默认行为 |
| --- | --- | --- |
| 负数与确认后更正 | `SAFE_DEFAULT_ACTIVE` | 拒绝正式负数；禁止直接覆盖 confirmed 记录。 |
| Excel/OCR 自审批 | `SAFE_DEFAULT_ACTIVE` | 上传者不能批准；要求另一财务账号。 |
| 病毒扫描不可用 | `SAFE_DEFAULT_ACTIVE` | 文件停留隔离区，不进入解析、OCR、预览或下载。 |
| 文件资源预算 | `SAFE_DEFAULT_ACTIVE` | 单文件硬上限 50 MiB；默认 OCR 处理 20 页；超限返回稳定错误，不靠资源耗尽超时。 |
| 外部 AI | `SAFE_DEFAULT_ACTIVE` | 真实数据外发关闭；本地失败转人工。 |
| 数据保留 | `SAFE_DEFAULT_ACTIVE` | 只生成 dry-run 计划，不自动删除；legal hold 永远优先。 |
| 报告结论 | `SAFE_DEFAULT_ACTIVE` | 未复核叙述只作为草稿；“合理/不合理”没有版本化规则时明确无法判断。 |
| 灾备与上线 | `SAFE_DEFAULT_ACTIVE` | 不宣称 RPO/RTO、真实 Staging、恢复或 production-ready 已通过。 |
