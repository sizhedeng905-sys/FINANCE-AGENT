# 周五演示能力限制

## OCR

- OCR 不在本次主演示链中。仓库已有合成 PDF、bbox、人工修订和双财务批准测试，但现场不启动真实 OCR 模型。
- 没有用已标注真实票据完成盲测，不能声明字段准确率、版面适配率或财务可接受率。
- 真实 OCR 仍需要 H04/H05 的真值集、指标门槛和人工签收；模型健康不等于识别准确。

## AI

- 本交付固定使用显式 Mock Provider，外部 Provider 为 `disabled`；不会连接 GitHub、互联网或真实模型。
- Excel 主线使用确定性列映射。AI 分类/映射只能作为待人工复核建议，不能审批、切换冻结模板、自动入账、执行 SQL 或计算财务金额。
- 老板页面的 AI 叙述不是主链验收条件。确定性 ReportSnapshot 才是数值事实源。
- 真实 Qwen/Paddle 等本地模型准确率、超时、显存和长期稳定性不由本交付证明。

## 报表与财务复核

- 自动化证明合成数据在当前固定查询口径下逐分一致，不等于 H06/H08 的真实日报/周报逐分对账已经签收。
- 经营日报只读取 `confirmed + actual + published` 记录并按币种分组；正式收入、成本、利润、冲销、关账和税务口径仍受人工决定约束。
- 公式单元格只使用已存在的缓存结果并显示 warning；系统不会运行 Excel 公式，也不会猜测缺失金额。
- 演示 seed 中的角色和密码只服务本地测试。正式职责分离、MFA/step-up 和人员授权仍需 H10。

## 环境与安全

- `demo:*` 命令只允许 loopback PostgreSQL 和精确数据库名 `finance_agent_test`，并拒绝 `NODE_ENV=production`。
- 演示 API 使用本地文件存储、basic 扫描、内存限流/执行门和 Mock Provider；这不是 production 的 S3、ClamAV、Redis、高可用或集中审计配置。
- `demo:reset` 会迁移、清理 E2E 前缀数据、恢复 demo seed 账号并重建合成 fixture，只能对隔离测试库运行。
- 当前未在目标云服务器、正式域名、TLS、对象存储、监控告警或灾备环境完成验收；这些仍是 H13-H16 的外部/人工门禁。
- 本包没有提交 `.env`、真实凭据、真实公司文件、上传物或模型权重。

## 性能与容量

- 仓库有 4,999 到 50,001 行边界测试与 49,999 行本地证据，但目标服务器容量、并发、磁盘水位和恢复时间尚未实测。
- 周五 fixture 只有 3 行，作用是稳定讲清业务完整性，不代表大文件现场性能。
- 本包不会在演示现场下载依赖、模型或漏洞数据库；依赖和浏览器必须提前准备。

## 当前能力声明

- `LOCAL_ENGINEERING_VERIFIED`：合成 Excel 双财务批准、正式记录、确定性日报与 Snapshot 链。
- `NOT_RUN`：三次连续人工现场演练、目标服务器 smoke、正式恢复演练。
- `REAL_SAMPLE_NEEDED`：真实 OCR、真实 Excel 映射、财务逐分对账和真实业务准确率。
- `NOT_PRODUCTION_READY`：本地演示成功不能作为上线、真实用户试运行或生产安全批准。
